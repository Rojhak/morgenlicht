# Specification: [Feature Name]

## 1. Context
- **Problem**: What are we solving?
- **Goal**: What is the desired outcome?

## 2. Requirements
- **Functional**: What must the system do?
- **Non-Functional**: Performance, security, accessibility constraints.
- **User Stories**: As a [user], I want to [action] so that [benefit].

## 3. Design
- **UI/UX**: Description or link to designs.
- **Technical Architecture**: Components, data flow, API changes.

## 4. Verification
- **Success Criteria**: How do we know we're done?
- **Test Plan**: Automated and manual tests.
