# Project Constitution

## 1. Principles
- **User First**: All changes must prioritize user value and experience.
- **Premium Aesthetics**: The UI must maintain a "10/10" premium feel. Warm, inviting, and professional.
- **Spec-Driven**: All non-trivial changes must be specified and planned before execution.

## 2. Technology Stack
- **Framework**: Next.js (App Router)
- **Styling**: Tailwind CSS (Semantic classes preferred over arbitrary values)
- **Language**: TypeScript (Strict mode)
- **Icons**: Lucide React

## 3. Coding Standards
- **Components**: Functional components with clear entry points.
- **Accessibility**: WCAG AA compliance is mandatory.
- **Performance**: Optimized images and code splitting.

## 4. Workflows
- **Planning**: Create a specification or implementation plan before coding.
- **Verification**: Verify changes functionally and visually (browser walkthroughs).
