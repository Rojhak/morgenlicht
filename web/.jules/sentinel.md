## 2024-05-24 - [Next.js Critical DoS and SSRF Vulnerabilities]
**Vulnerability:** Found multiple vulnerabilities in `next@15.1.9` including Denial of Service via server components, SSRF due to middleware redirect handling, and Cache Key Confusion.
**Learning:** Outdated frameworks, especially in SSR/SSG apps like Next.js, can expose critical infrastructure vulnerabilities (like DoS and SSRF) that aren't specific to application logic but rather framework behavior.
**Prevention:** Regularly audit (`pnpm audit`) and update core framework dependencies (e.g., Next.js) to patched versions, especially when new minor/patch versions are available.

## 2024-05-24 - [Resend SDK Silent Failure Pattern]
**Vulnerability:** The `resend.emails.send()` method from the Resend SDK v6.0.0+ does not throw an exception on failed email deliveries (e.g., invalid API key, malformed request). It instead returns an `{ error }` object which, if unhandled, leads to silent failures where the application assumes an email was sent successfully. Additionally, checking for the API key presence but returning a specific error message to the client leaks internal server configuration details.
**Learning:** SDKs may not always follow standard promise rejection patterns for API errors. Returning an `{ error }` object requires explicit checking. Furthermore, explicit error messages about missing configurations (like "E-Mail-Service nicht konfiguriert") expose internal infrastructure state to potential attackers, aiding in reconnaissance.
**Prevention:** Always check the documentation for specific SDK error handling patterns. For Resend, explicitly destructure and check the `{ error }` property from the response. Ensure that API routes return generic, non-revealing error messages (e.g., 'Interner Serverfehler') to the client while retaining full, detailed error logs on the server for debugging.
